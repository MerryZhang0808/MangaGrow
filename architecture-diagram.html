Constraints (Red Lines) - C01 ~ C15
C01 Env vars must use VITE_ prefix, read via import.meta.env
C02 Components (.tsx) must NOT call Gemini API directly
C03 All AI calls must use withRetry (min 3 retries)
C04 Model names from aiClient constants only, no hardcoding
C05 Panel = 1K, Avatar = 2K, no exceptions
C06 Avatar gen must include both photo + text description
C07 Style prompts from styleConfig.getStylePrompt() only
C08 Same ComicStyle + AspectRatio across all scenes in a batch
C09 5-step pipeline must execute in order, no skipping
C10 Quality gate max 2 retries, then degrade
C11 All JSON responses must pass JSON.parse() validation
C12 Photos must be compressed before API call (800px, 0.6)
C13 constants.ts SYSTEM_PROMPT must match Product-Spec.md
C14 Ref-image failure must fallback to text-only, no hard error
C15 Architecture.md changes must sync architecture-diagram.html

  storyService.generateStory()
        │  Step 1: analyzeInput ─────▶ structured events
        │  Step 2: generateOutline ──▶ story arc
        │  Step 3: reviewOutline ────▶ pass / retry(2x) / degrade
        │  Step 4: detailScripts ────▶ scene scripts with character visuals
        │  Step 5: checkConsistency ─▶ fix inconsistencies
        │
        ▼
  StoryOutput { scenes[], characters[], keyObjects[] }
        │
        ▼
  FOR EACH scene:
    imageService.generateSceneImage()
        │  inject: styleConfig prompts
        │  inject: character refs (avatar + sheet)
        │  inject: scene ref (user photo OR prev scene)
        │  ──▶ gemini-3-pro-image-preview [1K]
        │  ──▶ base64 image
        │
        ▼
  Update UI (DisplayPanel shows scene card)
