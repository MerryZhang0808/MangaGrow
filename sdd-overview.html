3 dev-constraints.md — 代码自动守卫（C01–C43）

作用范围：comic-growth-record/** 和 server/** 下所有编辑，自动生效，无需触发

🔐 环境 & AI 调用

C03 所有 Gemini 调用必须经 withRetry 包装（≥3次重试）
C04 模型名引用常量，不允许硬编码
C20 前端不允许存储或读取 API Key
C26 前端 services 不得直接 import @google/genai
C02 前端组件不直接调后端 API，必须经 services/ 层

🎨 角色 & 风格

C05 分镜图 1K（1024×1024），头像 2K（2048×2048）
C06 生成角色图必须同时传参考图+文字描述
C07 风格提示词必须从 styleConfig.getStylePrompt() 获取
C08 同一组分镜必须使用完全相同的风格参数
C14 有参考图方案失败时必须降级为纯文字方案

💾 数据 & API

C11 后端所有 AI 响应必须经 JSON.parse() 验证
C21 图片存储在 data/images/{type}/ 下
C23 前端展示用后端 URL，不用 base64 长存
C24 API 响应格式 { success, data?, error? }
C25 图片文件名必须使用 UUID

🔄 流程 & 文档

C09 故事管线 4 步必须按顺序执行，不允许跳步
C27 生成完成后显示「保存故事」按钮，不自动保存
C42 保存三步顺序：poster → storage → DB
C39 PDF 中文必须 Canvas 渲染，不直接调 jsPDF.text
C15 Architecture.md 变更时必须同步更新架构图 HTML
C44 修改 storyPipeline/imageGenerator/characterManager/API 路由时必须同步更新 agent-flow-diagram.html

5 一句话总结
文件 / Skill 角色 一句话
CLAUDE.md 协调者 Merry 守门员，确保 5 层不跳步，路由到正确 skill
product-spec-builder 废才 PM 把脑子里的浆糊逼成可执行的需求文档
architect 首席架构师 把需求变成 Claude Code 无法自由发挥的施工图纸

agent-flow-diagram.html 方法调用图 核心审核文件，代码变更时必须同步更新（storyPipeline/imageGenerator/characterManager/API）
