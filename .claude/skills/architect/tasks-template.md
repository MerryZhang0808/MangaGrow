# Tasks.md 输出模板

本模板用于生成结构完整的任务清单。生成时按照此结构填充内容。

---

## 模板结构

**文件命名**：Tasks.md

---

```markdown
# Tasks - [项目名称]

## 概述
<总共 X 个任务，其中 X 个 POC 任务，X 个开发任务>
<预计执行顺序和关键依赖说明>

---

## POC 任务（先验证再开发）

### [POC-01] <验证目标>
- **目的**：验证 <技术方案> 是否可行
- **方案引用**：Architecture.md → [对应章节]
- **验证方法**：
  1. <步骤1>
  2. <步骤2>
- **预期结果**：<什么结果算通过>
- **通过后**：将结果更新到 Architecture.md，解锁后续任务
- **状态**：⬜ 待验证

---

## 开发任务

### [T-01] <任务名称>
- **目的**：<实现什么功能>
- **方案引用**：Architecture.md → [对应章节]
- **修改文件**：
  - `<文件路径1>`：<做什么修改>
  - `<文件路径2>`：<做什么修改>
- **实现要求**：
  1. <具体要求1>
  2. <具体要求2>
- **约束**：
  - 必须遵守 Architecture.md 约束 C01, C02
- **验收标准**：
  - [ ] <标准1>
  - [ ] <标准2>
- **依赖**：无 / 依赖 [T-XX]
- **状态**：⬜ 待开发

## 状态说明

| 符号 | 含义 |
|------|------|
| ⬜ | 待开发/待验证 |
| 🔄 | 进行中 |
| ✅ | 已完成 |
| ❌ | 已否决/不再需要 |
| ⚠️ | 受上游变更影响，需重新审视 |

---

## 变更记录

| 日期 | 触发来源 | 变更内容 | 受影响任务 |
|------|---------|---------|-----------|
| YYYY-MM-DD | Architecture v1.1 (XX模块调整) | 新增 T-06，修改 T-03 验收标准 | T-03, T-06 |
```

---

## 写作要点

1. **POC 任务在前**：不确定的方案先验证，验证通过才开发
2. **方案引用**：每个任务必须引用 Architecture.md 的具体章节
3. **修改文件**：明确到文件级别，Claude Code 知道该改哪里
4. **验收标准**：checkbox 格式，完成后可打勾确认
5. **依赖关系**：明确前置任务，防止跳步
6. **粒度适中**：一个任务对应一个功能模块或一次有意义的代码提交，不要太大也不要太碎
