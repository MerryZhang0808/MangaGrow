3 H1 深度报告：跨分镜人物一致性
问题本质

AI 图片生成模型每次调用都是独立的，没有"记忆"。即使是同一个角色，不同分镜生成的外貌（发型、发色、服装、体型）几乎必然不同。这是 AI 绘本产品的最核心技术挑战。

迭代验证过程
v1.0 — 纯文字描述
仅靠 prompt 中的文字描述（如"黑色短发的小女孩"）来约束角色外貌。
失败 每张图的角色外貌几乎完全不同，文字约束力极弱。
v1.5 — 人物库 + Q版头像参考图
建立人物库系统，为每个角色生成 Q版头像。分镜生成时将头像作为参考图传入。
部分有效 有改善但不稳定。头像图片和文字描述分离，模型经常忽略头像。
v1.9 — 角色快照 + 链式生成
累积已生成分镜中角色的最佳画面作为后续参考。链式生成策略（前一分镜→下一分镜）。
改善 连续性提升，但首张到末张的漂移仍然明显。
v2.1 — 全链路五层修复 + 模型升级
系统性修复 5 个断裂点，从故事生成到图片生成端到端对齐。切换到 gemini-2.5-flash-image。

6 关键架构经验
端到端一致性需要全链路思维

人物一致性不是"图片生成 prompt 写好就行"的问题。需要从故事文本生成到图片输出全链路打通：

图 7 — 人物一致性全链路依赖关系
Multimodal Prompt 工程要点
  • 图文配对原则 — 参考图片必须紧贴其描述文字，不能分离放置
  • 优先级即排序 — Parts 数组中越靠前的内容对模型影响越大
  • 双重锚定 — 同时用图片（视觉）和文字（语义）约束同一对象，效果远优于单一通道
  • 负面标注 — 场景参考图需明确标注"不要参考此画面中的角色外貌"
  • 模型是前提 — 如果模型不支持 image-to-image 参考，所有 prompt 优化都是徒劳
临时角色一致性的已知局限

非人物库角色（AI 创造的临时角色）没有预设头像，只能靠文字描述 + 链式生成 + 角色快照。

更彻底的方案：在第一张分镜生成后，自动从画面中裁剪角色区域作为后续参考（待实现）。

穿刺结论

人物一致性方案已验证可行。通过全链路修复（故事管线对齐 + 双重锚定 + Parts 结构重构 + 模型升级），人物库角色的跨分镜一致性达到可用水平。

核心方法论：多模态 AI 的一致性问题不是单点 prompt 优化能解决的，需要"信息全链路无损传递"——从结构化数据、到文本描述、到图片参考，每个环节都不能有信息丢失。

后续优化方向：

  • 测试 gemini-3-pro-image-preview 是否有更好的参考图遵循能力
  • 自动角色裁剪 — 从首张分镜中提取角色区域作为临时角色的视觉锚点
  • 多角度参考表（Reference Sheet）效果验证
  • LoRA / IP-Adapter 等微调方案的可行性评估（长期）
