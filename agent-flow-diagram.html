故事生成管线 · storyPipeline.ts
4 步串行 AI 推理 + 第 4 步与标题生成并行 · 每步含 JSON Retry（最多 4 次）
入口：generateStory(input)
INPUT
StoryInput
用户输入聚合对象
字段 text（文字描述）
字段 imageAnalysis[]（图片分析结果）
字段 characters[]（人物档案）
字段 style（漫画风格）
字段 imageCount（分镜数约束）
Step 1
generateOutline(input)
分析核心事件，生成漫画故事大纲（起承转合 + 情感弧线）
⚠️ 人物规则两档：被提及角色强制出现 / 未提及角色按需
🔵 gemini-2.5-flash ↩ JSON Retry × 4 systemInstruction: STORY_SYSTEM_PROMPT
OUT scenes[] { beat, description, emotion }
OUT arc（情感弧线描述）
Step 2
reviewOutline(outline)
漫画编剧总监视角，审核大纲叙事推动力、因果连接、情感弧线、故事完整性
🔵 gemini-2.5-flash ↩ JSON Retry × 4 ⚠ 失败时降级（不阻断主流程）
OUT passed: boolean
OUT issues[]
OUT suggestions[]
⚡ passed=false → issues + suggestions 透传给 Step 3 作为修复指引；不重跑大纲，直接进 Step 3
Step 3
detailScripts(outline, input, reviewIssues?)
细化每格分镜脚本：description（50-100字画面描述）+ caption（30-50字叙述）+ emotionalBeat
同步提取 keyObjects 和 characterDefinitions 用于后续图片生成
v2.1：传入人物库视觉特征，要求分镜描述与人物库一致，characterDefinitions 复用人物库原文
🔵 gemini-2.5-flash ↩ JSON Retry × 4 systemInstruction: STORY_SYSTEM_PROMPT
OUT scripts[] { sceneNumber, description, caption, emotionalBeat }
OUT keyObjects[] { name, description }
OUT characterDefinitions[] { name, description }
⚡ PARALLEL EXECUTION — Promise.all()
Step 4a
checkConsistency(scripts, charDefs)
检查分镜间角色描述、场景连贯、时间线、物品一致性；有问题自动修正 description
🔵 gemini-2.5-flash ↩ JSON Retry × 4
OUT scripts[]（已修正）
OUT passed: boolean
‖
Step 4b
generateTitleInternal(text, imageAnalysis?)
生成 4-10 字故事标题：「{人名}+{核心事件}」格式，禁止情绪词开头
🔵 gemini-2.5-flash Fallback → text.slice(0,10)
OUT title: string（4-10字）
OUTPUT
StoryOutput
故事生成完成，返回结构化数据供前端展示
字段 title
字段 currentBatch[]（SceneScript）
字段 totalScenes
字段 keyObjects[]
字段 characterDefinitions[]
字段 storyOutline
年度总结模式（独立调用）
Export
generateYearlySummary(stories[])
根据多个 {title, inputText} 生成 300-500 字温馨成长总结，结尾含对孩子未来的期许
🔵 gemini-2.5-flash ↩ withRetry × 3 抛出异常 → 路由层降级
IN SummaryStoryItem[] { title, inputText }
OUT summary: string

人物创建管线 · characterAnalyzer.ts
3 步串行：外貌分析 → Q版头像生成 → 性别年龄识别
入口：createCharacterFull(name, imageData, mimeType)
INPUT
照片输入
家长上传的孩子照片
字段 name（人物名字）
字段 imageData（base64）
字段 mimeType
Step 1
analyzeCharacter(name, imageData, mimeType)
分析照片，生成 150-200 字超详细视觉描述：头发、默认服装（固定）、面部特征、配饰、体型。用于保证多画面人物一致性。
🔵 gemini-2.5-flash ↩ withRetry × 3 失败 → 降级 fallback 描述
IN 照片 inlineData + 分析指令
OUT description: string（150-200字）
Step 2
generateAvatar(name, imageData, mimeType, description, gender?, ageGroup?)
基于照片 + description 生成 Q版卡通头像（头身比 2:1~3:1，大眼萌系）
优先级：参考图识别度 > 可爱度 > 风格化。2K 分辨率，1:1 宫格。
🟣 gemini-3-pro-image-preview ↩ withRetry × 3 aspectRatio: 1:1 · imageSize: 2K
IN 照片 inlineData + description + 生成指令
OUT { data: base64, mimeType } → saveImage('avatars')
Step 3
detectGenderAge(imageData, mimeType)
从照片识别：性别（男/女/未知）、年龄段（婴儿/幼儿/儿童/少儿/成人/未知）
🔵 gemini-2.5-flash ↩ withRetry × 3 失败 → 默认"未知"
OUT gender: string
OUT ageGroup: string
OUTPUT
CharacterAnalysisResult
人物档案创建完成，存入 DB
字段 description
字段 avatarData + avatarMimeType
字段 gender
字段 ageGroup
独立导出方法
Export
analyzeCharacter(name, img, mime)
单独调用人物分析（API 路由 analyze-character 使用）
OUT description: string
·
Export
detectGenderAge(img, mime)
单独调用性别年龄检测（API 路由 detect-gender-age 使用）
OUT { gender, ageGroup }
·
Export
generateAvatar(name, img, mime, desc, ...)
单独调用头像重生成（API 路由 generate-avatar 使用，可更新 DB）
OUT { data, mimeType }

transcribeAudio(audioBase64, mimeType)
将家长的语音描述转录为文字，保留口语化风格

⚡ 多图并行示意（3张图为例）

‖
analyzeOne(img[1], 1)

analyzeOne(img[2], 2)

视频分析服务 · videoAnalyzer.ts v2.0
视频内容理解 + 关键帧提取，两步流程：TEXT_MODEL 分析内容 → IMAGE_MODEL 提取帧图片
导出：analyzeVideo
Step 1: 视频内容分析
1
analyzeVideoContent(videoBase64, mimeType)

🔵 gemini-2.5-flash ↩ withRetry × 3 responseMimeType: JSON

OUT { contentDescription, keyMoments[{ timestamp, description }] }
Step 2: 关键帧图片提取
2
extractKeyFrames(videoBase64, mimeType, contentAnalysis)

🟡 gemini-3-pro-image-preview ↩ withRetry × 3（每帧独立）
IN video inlineData + keyMoments
OUT VideoKeyFrame[] { index, timestamp, description, imageUrl }

分镜图生成 · imageGenerator.ts
角色身份优先 Parts 结构 · 头像+描述紧密配对 · 角色快照累积(v1.9) · 视觉引用失败自动降级为纯文本
入口：generateSceneImage(params)
INPUT
SceneGenerationParams
字段 script（分镜 description，50-100字）
字段 style（漫画风格）
字段 ratio（宽高比）
字段 characterContext
字段 objectContext
字段 referenceChars[]（CharacterImageRef，含 description）
字段 sceneReferenceImages[]
字段 characterSnapshots[] v1.9
字段 isUserPhoto?
Prompt
场景指令文本 v2.1
库角色描述已内联到 Parts 中与头像配对，此处仅含非库角色 + 场景 + 风格
[RULE] 严格按Q版头像绘制角色，服装优先画面描述
[OTHER CHARS] 非库角色的 characterContext（仅无头像的临时角色）
[SCENE] script（画面描述）
[OBJECTS] objectContext（关键物品）
[STYLE] getStylePrompt（风格参数）
[OUTPUT] 单格漫画，无水印/文字/多指，温馨可爱治愈风格
Parts
视觉参考注入策略 v2.1 角色身份优先
核心策略：① 角色身份优先建立（头像+描述配对）→ ② 角色快照 → ③ 场景参考（画风）→ ④ 场景指令
①人物参考 Q版头像 inlineData + 完整视觉描述（发型发色面部等）紧密配对
　多角度表 referenceSheet（如有）+ 标签
②角色快照 characterSnapshots[0] + 保持外观一致 v1.9
③场景参考 sceneReferenceImages[0] + 标签（参考画风，角色外观以头像为准）
④场景指令 scenePromptText（非库角色 + 场景描述 + 风格 + 质量）
Gen
executeGeneration(hasVisuals)
调用 Gemini 图像模型，imageSize: 1K
🟣 gemini-3-pro-image-preview ↩ withRetry × 3
🛡️ 有参考图失败 → 自动 Fallback 到纯文本模式（hasVisuals=false）再重试一次
OUT { data: base64, mimeType } → saveImage('scenes')
RES imageUrl: /api/images/scenes/xxx.png

通用重试机制 · gemini.ts → withRetry()
withRetry<T>(operation, retries=3)
指数退避 + 抖动：baseDelay = 1000ms × 2^i + random(0~1000ms)
4xx 错误不重试（429 除外）· 全部失败时抛出最后一次异常
Retry 1 ~1-2s
Retry 2 ~2-3s
Retry 3 ~4-5s
📝 storyPipeline 的 callAiWithJsonRetry 在此基础上额外增加 JSON 解析重试（最多4次），间隔 2s × 2^attempt

端到端流程 · 用户输入 → 漫画输出
完整数据流：从用户输入到最终漫画分镜图的全部处理步骤
👤 用户输入
前端 · App.tsx
用户在界面上的所有输入内容
文字 text（故事描述）
图片 inputImages[]（参考照片）
人物 characters[]（人物库角色）
风格 style（漫画风格）
Step 1
图片分析（可选）
如果用户上传了参考照片，调用 AI 分析图片内容
🔵 gemini-2.5-flash POST /api/ai/analyze-images
IN inputImages[]
OUT imageAnalysis[] { index, description }
⚡ 无图片时跳过此步骤
Step 2
故事生成管线 · storyPipeline.ts
4 步 AI 推理：大纲生成 → 质量审核 → 脚本细化 → 一致性检查（并行标题生成）
🔵 gemini-2.5-flash × 4-5 POST /api/ai/generate-story
IN text + imageAnalysis + characters + style
OUT scripts[]（分镜脚本）
OUT characterDefinitions[]（角色定义）
OUT title（故事标题）
子步骤： generateOutline → reviewOutline → detailScripts → checkConsistency + generateTitleInternal
Step 3
构建图片生成参数
前端组装每个分镜的图片生成请求参数
参数 script（分镜描述）
参数 style（漫画风格）
参数 characterContext（角色描述）
参数 referenceCharIds[]（人物库ID）
参数 sceneReferenceImages[]（连续性参考）
💡 人物库角色：characterContext 传递完整视觉描述 + 头像引用指令（双重锚定）；非人物库角色：传递 AI 生成的外貌描述
Step 4
加载人物库头像
后端根据 referenceCharIds 从数据库加载 Q 版头像图片
server/routes/ai.ts → generate-image
IN referenceCharIds[]
OUT referenceChars[] { name, description, avatarData, avatarMimeType }
⚡ 无人物库角色时 referenceChars = []
Step 5
分镜图生成 · imageGenerator.ts
调用 Gemini 图像生成 API，结合文字描述和参考图片生成分镜图
🟣 gemini-3-pro-image-preview POST /api/ai/generate-image
IN script + style + characterContext
IN referenceChars[]（Q版头像）
IN sceneReferenceImages[]（连续性参考）
OUT imageUrl（生成的分镜图）
Parts 结构（角色身份优先 v2.1）：
① [Q版头像图片] + 完整视觉描述（发型发色面部等，紧密配对）
② [角色快照（可选）] + "保持外观一致"
③ [场景参考帧（可选）] + "参考画风，角色以头像为准"
④ [场景指令] RULE + OTHER_CHARS + SCENE + STYLE + OUTPUT
Step 6
循环生成所有分镜
对每个分镜重复 Step 3-5，使用角色快照机制保持人物外观一致性
循环 scenes[0] → scenes[1] → ... → scenes[n]
连续性 sceneReferenceImages（前一分镜）
📸 角色快照机制 (v1.9)
输入：characterSnapshots[0]（该角色在已生成分镜中的最佳参考图）
标签：「[角色一致性参考] 保持与该角色在之前分镜中的外观一致。」
更新：每生成分镜后，将 imageUrl 存入 characterSnapshots[charId]，作为后续分镜的参考
✅ 最终输出
漫画分镜组
完整的漫画故事，包含标题和所有分镜图
输出 title（故事标题）
输出 scenes[] { imageUrl, script, caption }
输出 keyObjects[]（关键物品）
📊 关键数据流
文字描述流向
user.text → imageAnalysis → storyPipeline → scripts[] → imageGenerator.prompt
人物库角色流向
characters[] → referenceCharIds[] → DB.query(含description) → referenceChars[] → 头像+描述配对注入 imageGenerator.parts
图片连续性流向
scene[n-1].imageUrl → sceneReferenceImages → imageGenerator → scene[n].imageUrl
角色快照流向 (v1.9)
scene.imageUrl → characterSnapshots[charId] → imageGenerator.parts（重新生成时也传入相邻分镜作为快照）
